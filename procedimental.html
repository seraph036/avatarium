Programação Procedimental

  • O paradigma procedimental estrutura o código como uma série de procedimentos, que tendem a modificar variáveis globais e o state, que se torna comum ao aplicativo como um todo, ou, pelo menos, a uma grande parte dele.
  • Por um lado, essa organização torna o código direto e de rápida confecção, uma vez que o escopo de variáveis ou a acessibilidade do state tendem a não se tornarem impedimentos a quaisquer implementações.
  • Por outro lado, essa abordagem pode introduzir, caso o programador não seja cuidadoso com os acessos às variáveis, problemas de comportamento imprevisível ou indesejado, especialmente no caso de funções async, código concorrente ou paralelo, dentre outros.
  • O acesso descontrolado pode permitir que variáveis sejam modificadas enquanto estão em utilização por outras partes do programa, ou que funções assíncronas modifiquem o state, antes que isso seja desejado. Isso tende a acontecer em grandes codebases, somente se houver falta de cuidado nas condições de acesso aos dados.

Dicionário de Termos

State
=> É a coleção de dados coletados pelo programa durante sua execução, sejam eles por input do usuário, sejam eles por um banco de dados criado previamente ao runtime do programa. Geralmente, o termo é utilizado para denominar a coleção de variáveis globais ou de escopo alto presentes em um programa.
