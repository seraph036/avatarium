Programação Lógica

  • O paradigma lógico é um padrão declarativo de programação, no qual o aplicativo é construído como uma sequência de predicados da lógica formal, com a computação sendo performada por meio da interpretação lógica dessas expressões.
  • Esse método é bastante utilizado em áreas que requerem definições ou restrições extensas, como processamento de dados e machine learning, processamento de linguagem natural, gerenciamento de bancos de dados, análise preditiva, dentre outros.
  • Um campo muito importante para esse paradigma é a Prova Automática de Teoremas, que foi imprescindível para o surgimento da Ciência da Computação como um todo, e permite que programas deduzam e provem teoremas matemáticos.
  • Os componentes principais de uma linguagem lógica são os facts, as rules e as queries. Esses definem o fluxo e os métodos do aplicativo, seguindo os padrões da lógica formal.

=> Linguagens que priorizam esse paradigma:

Dicionário de Termos

PAT
=> A Prova Automática de Teoremas é uma forma de raciocínio automatizado, em que há tentativa de provar um teorema matemático por uso de um computador. Embora não seja possível provar todos os teoremas, como demonstrado pelo Teorema de Incompletude de Gödel e pelo Problema P vs NP, esse campo é extremamente importante para tratar problemas decidíveis, como demonstrado pelo Teorema da Completude de Gödel.

Dicionário de Termos

Facts
=> São sentenças verdadeiras, utilizadas como base para a dedução de outras inferências. Formam a base lógica do código nesse paradigma.

Dicionário de Termos

Rules
=> É uma formula lógica que descreve uma relação entre dois valores, nesse caso, dois facts. A partir dos valores booleans inferidos a partir dessas relações, o fluxo do código é definido, bem como quais métodos declarados serão aplicados.

Dicionário de Termos

Queries
=> É um pedido de informação ou uma condição específica que o programa lógico tenta satisfazer ou completar utilizando os métodos lógicos declarados. Tipicamente, consiste em uma 'pergunta' sobre facts e rules declarados no código, buscando indagar sobre a validade de relações entre dados, para estruturar o dataflow do programa.
